# Деплой на Хостинг Україна (ukraine.com.ua)

Цей каталог містить інструкції для публікації **статичного сайту** (Vite + React) після збірки. У панелі хостингу зазвичай використовується каталог **`public_html`** (або `www`, `httpdocs` — залежить від тарифу; уточніть у підтримки або в документації кабінету).

## Що збирається

Після `npm run build` у корені проєкту з’являється папка **`dist/`** — це готовий продакшен-артефакт (HTML, JS, CSS, `robots.txt`, `sitemap.xml`, статичні сторінки з `public/` тощо). Скрипт збірки також оновлює **`lastmod`** у `dist/sitemap.xml`.

## Підготовка на своєму ПК

1. У корені репозиторію: `npm install` (або `pnpm install`, якщо використовуєте pnpm).
2. Збірка: **`npm run build`**.
3. Перевірка локально (опційно): `npm run preview` і відкрити `http://127.0.0.1:4173`.

## Що завантажувати на сервер

Завантажуйте **вміст** папки `dist/` (не саму папку `dist` як один рівень зайвої вкладеності, якщо хочете сайт на корені домену), тобто файли на кшталт:

- `index.html`
- папки `assets/` з хешованими JS/CSS
- `favicon.svg`, `robots.txt`, `sitemap.xml`, `og-image.png`
- `privacy.html`, `terms.html` (якщо є в збірці)

У **`public_html`** має лежати `index.html` у корені, щоб відкривався головний URL `https://ваш-домен/`.

### Швидка копія для архіву (Windows)

З каталогу `hosting-ukraine-export` виконайте:

```bat
copy-dist.bat
```

Скрипт скопіює вміст `../dist` у **`site-export/`** (ця папка в `.gitignore`). Далі можна заархівувати `site-export` і завантажити через файловий менеджер або FTP/SFTP.

## HTTPS і домен

У кабінеті хостингу підключіть SSL (Let’s Encrypt або наданий сертифікат) і прив’яжіть домен до каталогу з сайтом.

## URL у проєкті (canonical, OG, JSON-LD)

У **`index.html`** задано абсолютні URL для **`https://cloud-pro.com.ua/`** (canonical, Open Graph, Twitter, структуровані дані). Якщо сайт виставляється на **інший домен**, перед продакшен-збіркою оновіть ці посилання та перезберіть проєкт — інакше пошуковики й соцмережі можуть показувати старий домен.

## Сумісність з Apache (якщо потрібні «глибокі» посилання)

Зараз у проєкті немає клієнтського роутингу з окремими шляхами; достатньо стандартної віддачі `index.html`. Якщо пізніше з’являться маршрути React Router і прямі заходи на URL крім `/`, може знадобитися **`public_html/.htaccess`** з правилами `mod_rewrite`, щоб несуществуючі файли перенаправляли на `index.html`. Уточніть у підтримки, чи увімкнено `mod_rewrite` на вашому тарифі.

## Перевірка після завантаження

- Відкривається головна сторінка по HTTPS.
- Працюють стилі та скрипти (немає 404 у вкладці Network).
- `https://ваш-домен/robots.txt` та `https://ваш-домен/sitemap.xml` віддаються коректно.

Детальний чекліст — у файлі **`CHECKLIST.txt`**.
